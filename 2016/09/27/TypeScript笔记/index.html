TypeScript装饰器

2016-09-27 • 分类: 笔记 • 标签: typescript, 笔记

装饰器是ES7将会提供的功能，目前TypeScript已提供支持，为了避免编译警告，需要在tsconfig.json中设置 compilerOptions字段中的 experimentalDecorators 为 true，如果需要编译器生成 元数据 还需要指定 compilerOptions字段中的 emitDecoratorMetadata 为 true。

类装饰器

类装饰器仅接受一个参数，即类的构造函数。

由于装饰器在类声明后被调用，所以装饰器内修改类的prototype会覆盖类的声明

类装饰器可以返回一个值，该值会替换掉类的构造函数

方法装饰器

注意:

如果代码输出目标版本小于ES5，Property Descriptor将会是undefined。

装饰器返回的值也会被忽略

访问符装饰器

访问符装饰器与方法装饰器相同，不过值得注意的一点事，访问符装饰器不能给同名的 get 和 set 设置不同的装饰器，因为本质上来讲，get 与 set 公用一个 属性名与属性描述符。

属性装饰器

属性属于动态赋值的内容，所以在编译期间拿不到属性的属性描述符，所以装饰器仅接受两个参数。

静态属性虽然可以在编译期间拿到属性描述符，但可能是为了一致性，TypeScript 静态属性装饰器也没有属性描述符这个参数

参数装饰器

参数装饰器仅可以用于类方法的参数，不可以用在函数上。
